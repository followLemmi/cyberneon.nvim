# Contributing to cyberdyne-port.nvim

Thank you for your interest in contributing to cyberdyne-port.nvim!

## How to Contribute

### Reporting Bugs

If you find a bug, please create an issue with:
- A clear description of the problem
- Steps to reproduce
- Your environment (OS, Neovim version, terminal)
- Screenshots if applicable

### Suggesting Features

Feature suggestions are welcome! Please create an issue describing:
- The feature you'd like to see
- Why it would be useful
- Any examples from other themes

### Submitting Pull Requests

1. Fork the repository
2. Create a new branch for your feature/fix
3. Make your changes
4. Test your changes thoroughly
5. Commit with a clear message
6. Push to your fork
7. Open a pull request

### Code Style

- Follow existing code style
- Use meaningful variable names
- Add comments for complex logic
- Keep color definitions in the main palette

### Color Guidelines

When adding new highlight groups:
- Use colors from the existing palette
- Ensure good contrast and readability
- Test in both light and dark terminals
- Consider colorblind accessibility

### Testing

Before submitting:
- [ ] Test with different file types (Lua, Python, JavaScript, etc.)
- [ ] Check common plugins (Telescope, Neo-tree, etc.)
- [ ] Verify LSP diagnostics are visible
- [ ] Test with TreeSitter enabled

## Questions?

Feel free to open an issue for any questions about contributing!

Thank you for making cyberdyne-port.nvim better!
